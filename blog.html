Discuss the scope of var, let, and const?

In JavaScript var, let and const are used to declere variables. But there are some major difference among them. let's discuss the metter,

var

1.before modern JS or ES-6 var was used to decler variables.

2.var is function-scoped.

3.If var declered outside a function then it become globally scopped, can be access the entire program.

4.Never abide by the law of block scope like let or const do.

5.var can be re-declered and re-asigned, He never minds.

6.if we declere a variable using var it become hoisted.

let

1.After ES-6 we are introduced with let.Like var let is also used to declere variables.

2.Variables declared with let are block-scoped. This means they are only accessible within the block in which they are defined

3.We can't re-declere a variable using let but we can re-assign.

4.Variables declered using let will be never hoisted.

5.Accessing a let variable before its declaration will give an error.

const

1.Like var and let, const is used to declere variable.

2.Like let, Variables declared with const are block-scoped. This means they are only accessible within the block in which they are defined

3.We can't re-declere and re-assign a variable using const.

4.Variables declered using const will be never hoisted.

5.Accessing a const variable before its declaration will give an error.

6.const is used to declare variables that should not be reassigned after their initial value is assigned

What is the use cases of null and undefined?

In JavaScript, null and undefined both are falsy value, that represent the absence of a value or the lack of a meaningful value, but they have some different use cases:

null

1.null is a primitive value that represents the intentional absence of any object value.

2.It is often used when we want to explicitly indicate that a variable or object property should have no value or that it is empty.

3.We can assign null to a variable or an object property to indicate that it has no value.

undefined

1.undefined is a primitive value that represents the absence of a value when a variable or object property has not been assigned a value.

2.It is often the default value of uninitialized variables and function parameters.

3.It can also be used to check if a variable has been defined or if a property exists in an object.

What do you mean by REST API?

REST API, which stands for Representational State Transfer Application Programming Interface, is an architectural style and set of constraints for designing and interacting with web services. It is one of the most popular and widely used approaches for building APIs (Application Programming Interfaces) on the World Wide Web.
